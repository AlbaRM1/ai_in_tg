# Universal LLM Telegram Bot

Модульный production-ready Telegram бот для работы с различными LLM через OpenAI-совместимые эндпоинты. Каждый пользователь подключает свой собственный API endpoint.

## 🎯 Основные возможности

- ✅ **Универсальный LLM-клиент**: каждый пользователь подключает свои OpenAI-совместимые эндпоинты (`base_url` + `api_key`). Список моделей подтягивается прямо из эндпоинта, отображается **с пагинацией** (по 8 на странице), поддерживаются **избранные модели** (⭐).
- ✅ **Регистрация по токенам**: доступ к боту только по одноразовому токену. Админ генерирует токены командой `/generate_token`; за фильтрацию доступа отвечает **auth middleware**.
- ✅ **Форум-топики как сессии**: каждый топик супергруппы — это **отдельный диалог** (как «New Chat» в ChatGPT). Глобальные endpoint и модель из `/settings` служат default для новых сессий. В существующем топике команда `/model` явно меняет модель либо пару endpoint+model только этой сессии, не затрагивая глобальные настройки и историю. Если закреплённый endpoint удалён, `/model` предлагает recovery через другой endpoint владельца. Бот отправляет и **закрепляет инфо-сообщение** о начальной модели, а также **автоматически переименовывает топик** по первому сообщению.
- ✅ **Мультимодальность**:
  - Изображения (vision через base64 data URL — формат OpenAI Vision)
  - Документы (извлечение текста из PDF и текстовых файлов: `.txt`, `.md`, `.py`, `.json`, `.csv` и др.)
  - Текст с caption
- ✅ **Стриминг ответов**: сообщение динамически обновляется по мере генерации — «💭 Думаю…» постепенно наполняется текстом (троттлинг обновлений ~1.5 с), с typing-индикатором.
- ✅ **🔎 Веб-поиск в интернете (Tavily)**: модель **сама решает, когда искать** актуальную информацию (погода, новости, курсы, события) через **tool calling** и **агентный цикл**. Во время поиска показывается индикатор «🔎 Ищу в интернете…», затем стримится финальный ответ. Подробнее — в разделе [🔎 Веб-поиск (Tavily)](#-веб-поиск-tavily).
- ✅ **Форматирование**: конвертация Markdown-ответа модели в **Telegram-HTML** (жирный/курсив/зачёркнутый, инлайн-код и блоки кода с подсветкой, ссылки, списки, заголовки), expandable blockquote для thinking-блоков (`<think>…</think>`).
- ✅ **Агрегация входящих сообщений**: длинные/пересланные тексты, которые Telegram режет на части, и **альбомы** (media-group) собираются в **один запрос → один ответ** через дебаунс (~1.5 с тишины).
- ✅ **Безопасная нарезка длинных ответов**: ответы больше 4096 символов разбиваются на несколько сообщений **без разрыва HTML-тегов** (теги закрываются в конце части и переоткрываются в начале следующей), с анти-флуд задержкой и graceful fallback на plain text при `MESSAGE_TOO_LONG`/невалидном HTML.
- ✅ **Три варианта БД**: локальный **PostgreSQL** (+ скрипт `setup_db.sh`), локальный **SQLite** (запасной вариант), внешняя **managed-PostgreSQL** (Neon/Supabase) с SSL и pooler-совместимостью и понятной диагностикой сетевых ошибок (Supabase IPv6 → Session Pooler).
- ✅ **Управление контекстом**: sliding window с лимитом токенов (`MAX_CONTEXT_TOKENS`, по умолчанию 100k), model-aware подсчёт токенов через `litellm`.
- ✅ **Актуальные дата и время для модели**: каждый сетевой LLM-запрос получает динамический system-блок с локальным временем из `APP_TIMEZONE`; значение не сохраняется в БД.
- ✅ **Production-ready**:
  - Async everywhere (aiogram 3.x, SQLAlchemy 2.0 async, asyncpg / aiosqlite)
  - Структурированное логирование
  - Обработка ошибок (таймауты, rate limits, retry)
  - Middleware-архитектура (Database + Auth)
  - Задел под шифрование API-ключей (Fernet-ready; сейчас — pass-through заглушки, см. раздел «Расширенная настройка»)

## 🔎 Веб-поиск (Tavily)

Бот умеет **искать актуальную информацию в интернете** через провайдера [Tavily](https://tavily.com) с помощью **tool calling** и **агентного цикла**. Модель **сама решает**, когда ей нужно обратиться к поиску (например, при вопросах о свежих событиях, погоде, новостях, курсах валют, текущих фактах или темах, в которых она не уверена), получает результаты и формирует финальный ответ на их основе.

### Как это работает

- В запрос к модели передаётся инструмент `web_search` (`tool_choice="auto"`).
- Если модель решает вызвать поиск, бот выполняет запрос к Tavily REST API, возвращает результаты модели и продолжает диалог (**агентный цикл**, до 4 итераций).
- Во время поиска статус-сообщение показывает индикатор `🔎 Ищу в интернете: <запрос>…`.
- После получения результатов бот **стримит финальный ответ**: индикатор поиска сразу сменяется постепенно наполняющимся текстом (тот же динамический стриминг, что и при обычном ответе).
- Готовый ответ форматируется в Telegram-HTML и при необходимости нарезается на части как обычно.

### Как включить

1. Зарегистрируйтесь **бесплатно** на [tavily.com](https://tavily.com) и получите API-ключ (вида `tvly-...`).
2. Добавьте ключ в ваш `.env`:

   ```env
   TAVILY_API_KEY=tvly-...
   ```

3. Перезапустите бота. В логах при старте появится строка `Web search: enabled (Tavily key set)`.

### Важно

- Веб-поиск работает **только на эндпоинтах/моделях с поддержкой function/tool calling**. Если эндпоинт не поддерживает инструменты, бот **автоматически откатывается** на обычный ответ без поиска (graceful fallback) — пользователь всё равно получит ответ.
- Если `TAVILY_API_KEY` **не задан**, фича просто выключена — бот работает как обычно (обычный стриминг без инструментов).

### 2. Автонастройка базы данных

**Простой способ (рекомендуется):**

Используйте скрипт автоматической настройки PostgreSQL:

```bash
./setup_db.sh
```

Скрипт автоматически:
- Проверит установку и запуск PostgreSQL
- Создаст пользователя `ai_in_tg_user` и базу `ai_in_tg`
- Сгенерирует безопасный пароль
- Обновит `DATABASE_URL` в файле `.env`

**Кастомизация параметров:**

```bash
# Переопределить имя базы и пользователя
DB_NAME=mybot DB_USER=myuser ./setup_db.sh

# Задать собственный пароль
DB_PASSWORD=secret ./setup_db.sh
```

**Ручная настройка (альтернатива):**

Если предпочитаете настраивать вручную:

```bash
# Создайте базу данных
sudo -u postgres psql
postgres=# CREATE DATABASE llm_bot;
postgres=# CREATE USER llm_user WITH PASSWORD 'your_secure_password';
postgres=# GRANT ALL PRIVILEGES ON DATABASE llm_bot TO llm_user;
postgres=# \q
```

**Запасной вариант: SQLite (без PostgreSQL)**

Если PostgreSQL не заводится или недоступен, можно использовать локальный SQLite как запасной вариант:

1. В файле `.env` укажите SQLite-строку подключения:
   ```env
   DATABASE_URL=sqlite+aiosqlite:///./bot.db
   ```

2. Скрипт `./setup_db.sh` НЕ нужен — таблицы создаются автоматически при первом запуске `python main.py`

3. Файл `bot.db` создастся в корне проекта (он в `.gitignore` и не попадёт в репозиторий)

**⚠️ Важно:** SQLite — крайний случай для локальной разработки или тестирования. Для продакшена рекомендуется PostgreSQL (лучшая производительность, поддержка JSONB, concurrent access).

### 2.1. Хостинг БД: Managed-PostgreSQL (Neon / Supabase)

**Зачем это нужно:**

Бесплатные хосты приложений (Render Free Tier, Railway Trial, Fly.io, и т.д.) могут **внезапно удаляться** при бездействии или истечении пробного периода. При использовании локальной БД на хосте все данные будут потеряны.

**Managed-PostgreSQL** (Neon, Supabase) решает эту проблему:
- ✅ Данные живут **вне хоста приложения** (в облачной БД)
- ✅ При переезде на новый хост достаточно **передеплоить приложение** с тем же `DATABASE_URL`
- ✅ Бэкапы и восстановление **не требуются**
- ✅ Бесплатные тиры: Neon (0.5 ГБ, 10 часов compute/месяц), Supabase (500 МБ, 2 ГБ bandwidth)

#### Получение бесплатной БД

**Вариант 1: Neon (рекомендуется)**

1. Зарегистрируйтесь на [neon.tech](https://neon.tech)
2. Создайте новый проект (выберите регион ближе к вашему хосту)
3. Скопируйте **Pooled connection string** (важно для serverless-окружений)
4. Приведите строку к формату проекта:
   ```env
   # Исходная строка от Neon:
   # postgresql://user:password@ep-xxxxx.eu-central-1.aws.neon.tech/dbname?sslmode=require
   
   # Формат для проекта (добавьте +asyncpg):
   DATABASE_URL=postgresql+asyncpg://user:password@ep-xxxxx.eu-central-1.aws.neon.tech/dbname?sslmode=require
   ```
5. Готово! Код автоматически обработает SSL и настроит совместимость с pooler.

**Вариант 2: Supabase**

⚠️ **ВАЖНО про IPv6 / Network is unreachable:**

На бесплатном тарифе Supabase **прямое подключение** (Direct connection, эндпоинт `db.<project-ref>.supabase.co:5432`) доступно **ТОЛЬКО по IPv6**. Если на вашей машине нет исходящего IPv6-маршрута (большинство обычных сетей и VPS), вы получите ошибку:

```
OSError: [Errno 101] Network is unreachable
```

**✅ РЕШЕНИЕ: Используйте Session Pooler (доступен по IPv4)**

1. Зарегистрируйтесь на [supabase.com](https://supabase.com)
2. Создайте новый проект (выберите регион ближе к вашему хосту)
3. Перейдите в **Project Settings → Database → Connection string**
4. **ОБЯЗАТЕЛЬНО** выберите вкладку **«Session pooler»** (НЕ «Direct connection»!)
5. Скопируйте строку подключения — она имеет вид:
   ```
   postgresql://postgres.<project-ref>:[YOUR-PASSWORD]@aws-0-<region>.pooler.supabase.com:5432/postgres
   ```
   
   **Обратите внимание:** пользователь в pooler-строке имеет вид `postgres.<project-ref>` (с точкой), а не просто `postgres`.

6. Приведите строку к формату проекта (добавьте `+asyncpg` и `?sslmode=require`):
   ```env
   DATABASE_URL=postgresql+asyncpg://postgres.<project-ref>:<password>@aws-0-<region>.pooler.supabase.com:5432/postgres?sslmode=require
   ```

7. Готово! Код автоматически обработает SSL и настроит совместимость с pooler (`statement_cache_size=0` уже выставлен).

**Пример полной строки:**
```env
# Для проекта с ref "abcdefghijklmnopqrst" в регионе eu-central-1:
DATABASE_URL=postgresql+asyncpg://postgres.abcdefghijklmnopqrst:your_password@aws-0-eu-central-1.pooler.supabase.com:5432/postgres?sslmode=require
```

#### Важные детали

- **SSL обязателен**: managed-провайдеры требуют TLS-соединение (`?sslmode=require` в URL)
- **Pooler-совместимость**: код автоматически выставляет `statement_cache_size=0` для корректной работы с pgBouncer (Neon pooler)
- **Автовосстановление соединений**: `pool_pre_ping=True` и `pool_recycle=300` защищают от разрывов при засыпании бесплатного тира
- **Локальный PostgreSQL не ломается**: если в URL нет `sslmode`, SSL не используется

#### Миграция на новый хост

При переезде на новый бесплатный хост приложения:

1. Задеплойте приложение на новый хост (Render / Railway / Fly.io)
2. В переменные окружения укажите **тот же** `DATABASE_URL` от Neon/Supabase
3. Все данные (пользователи, эндпоинты, история чатов) автоматически доступны
4. **Бэкапы не нужны** — данные уже в облаке!

### 3. Конфигурация

Скопируйте `.env.example` в `.env` и заполните:

```bash
cp .env.example .env

```

Минимальная конфигурация:

```env

ADMIN_TELEGRAM_ID=123456789                       # Ваш Telegram ID
DATABASE_URL=postgresql+asyncpg://llm_user:your_secure_password@localhost:5432/llm_bot
```

#### Полный список переменных окружения

| Переменная | Обязательна | Значение по умолчанию | Описание |
|------------|:-----------:|-----------------------|----------|
| `BOT_TOKEN` | ✅ | — | Токен Telegram-бота от [@BotFather](https://t.me/BotFather) |
| `ADMIN_TELEGRAM_ID` | ✅ | — | Telegram ID администратора (кто может выпускать токены через `/generate_token`) |
| `DATABASE_URL` | ✅ | — | Строка подключения к БД. Схема **обязательно** с драйвером: `postgresql+asyncpg://…` или `sqlite+aiosqlite:///./bot.db` |
| `APP_TIMEZONE` | ❌ | `Asia/Yekaterinburg` | IANA-часовой пояс динамических даты и времени в system-контексте каждого LLM-запроса. При невалидной зоне используется UTC с предупреждением в логе |
| `MAX_CONTEXT_TOKENS` | ❌ | `100000` | Лимит токенов контекста (sliding window) |
| `TAVILY_API_KEY` | ❌ | — | **Опционально — веб-поиск.** Ключ Tavily (`tvly-…`). Если не задан — веб-поиск отключён, бот работает как обычно. Получить бесплатно: [tavily.com](https://tavily.com) |
| `ENCRYPTION_KEY` | ❌ | — | Опционально. Fernet-ключ для шифрования API-ключей. Сейчас шифрование — заглушка (pass-through); подробнее см. «Расширенная настройка» |

> ⚠️ **Важно про `DATABASE_URL`:** схема URL обязательно должна содержать драйвер — `postgresql+asyncpg://` (PostgreSQL) или `sqlite+aiosqlite://` (SQLite). Голая схема `postgresql://` заставит SQLAlchemy искать синхронный `psycopg2`, которого в проекте нет. Проект автоматически подставит `+asyncpg`, но лучше указать драйвер явно.

## 🛠️ Использование

### Для администратора

1. **Генерация токена регистрации**:
   - Отправьте боту `/generate_token` в личке
   - Получите одноразовый токен
   - Передайте токен пользователю

### Для пользователя

1. **Регистрация**:
   - Отправьте боту `/start <токен>` в личке
   - Токен используется один раз и становится неактивным

2. **Настройка эндпоинта**:
   - Отправьте `/settings` в личке с ботом
   - Выберите "🔌 Мои эндпоинты" → "➕ Добавить эндпоинт"
   - Введите название (например, "OpenAI")
   - Введите URL (например, `https://api.openai.com`)
   - Введите API ключ (он будет зашифрован)
   - Бот автоматически проверит эндпоинт и получит список моделей
   - Активируйте эндпоинт в списке

3. **Выбор модели по умолчанию**:
   - В `/settings` выберите "🤖 Выбор модели"
   - Выберите модель из списка — она станет default для новых сессий
   - Можете добавить модели в избранное (⭐)
   - Изменение default не переключает уже существующие топики

4. **Использование в топиках**:
   - Создайте **supergroup** в Telegram
   - В настройках группы включите **Topics** (форум-режим)
   - Добавьте бота в группу и дайте **права администратора**
   - Для автопереименования топиков боту нужно право **«Управление темами» (`can_manage_topics`)** — без него бот ответит, но не сможет переименовать топик и корректно закрепить инфо-сообщение (в этом случае он просто пропустит эти действия — graceful fallback)
   - Создайте топики для разных чатов
   - Пишите в топиках — бот ответит со **стримингом**
   - Каждый топик = **отдельная сессия** с собственной историей (до `MAX_CONTEXT_TOKENS`)
   - Модель **фиксируется за сессией** при первом сообщении; бот отправит и закрепит инфо-сообщение вида «🤖 Модель этого чата: …»
   - Команда `/model` внутри существующего топика показывает текущую модель, endpoint, избранные модели этого endpoint и полный список с пагинацией по 8
   - В обычном меню можно сменить только модель либо нажать «Сменить endpoint», выбрать owner-scoped endpoint и затем его модель. Нажатие модели во втором flow является явным подтверждением одновременной смены endpoint+model
   - Если endpoint сессии удалён (`endpoint_id=NULL`), история и прежнее имя модели сохраняются, а `/model` показывает доступные endpoint пользователя; глобально активный endpoint отмечен ✅. Если endpoint-ов нет, меню направляет в личные `/settings`
   - Если существующий endpoint временно не отдаёт список моделей, он не считается удалённым: доступны «Повторить», «Выбрать другой endpoint» и «Закрыть», БД автоматически не меняется
   - Любой выбор через `/model` меняет только текущий топик. Глобальные `active_endpoint`/`active_model` и история диалога не изменяются; новая пара используется со следующего запроса
   - Перепривязка endpoint+model выполняется атомарно и отклоняет callback из устаревшего меню; исчезнувшие endpoint/модель требуют обновить выбор
   - Топик автоматически переименовывается по первому сообщению
   - Поддерживаются: текст, изображения (фото), документы (PDF, текстовые файлы), а также **альбомы** и длинные/пересланные сообщения (собираются в один запрос)

## 🔧 Расширенная настройка

### Подключение шифрования API-ключей

Сейчас функции `encrypt`/`decrypt` в [`app/utils/crypto.py`](app/utils/crypto.py:1) — **заглушки (pass-through)**: ключи хранятся в БД как есть. Интерфейс уже готов под подключение `cryptography.Fernet`. Для настоящего шифрования:

1. Сгенерируйте Fernet ключ:

```python
from cryptography.fernet import Fernet
print(Fernet.generate_key().decode())
```

2. Добавьте в `.env`:

```env
ENCRYPTION_KEY=your_fernet_key_here
```

3. Раскомментируйте код в `app/utils/crypto.py`

4. API ключи автоматически шифруются при добавлении эндпоинтов

### Настройка времени и лимита контекста

```env
# IANA timezone для динамического system-контекста каждого LLM-запроса
APP_TIMEZONE=Asia/Yekaterinburg

MAX_CONTEXT_TOKENS=200000  # Пример явной настройки; встроенный default — 100k
```

Дата и время вычисляются заново непосредственно перед сетевым вызовом LLM, включая
внутренние раунды web-search agent и LLM-сжатие. Runtime-блок содержит локальную
дату, время до секунд, день недели, UTC offset, имя зоны и ISO 8601; он не хранится
в БД и не добавляется навсегда к базовому system prompt. Неизвестная IANA-зона
безопасно заменяется на UTC и фиксируется предупреждением в логе.

```python
logging.basicConfig(level=logging.DEBUG)  # Для отладки
```

## 🐛 Troubleshooting

### Bot не отвечает в топиках

- Убедитесь, что группа в режиме forum (Topics включены)
- Бот должен быть администратором с правами на сообщения
- Пишите именно в топиках, не в General
- Проверьте, что вы зарегистрированы и настроили эндпоинт

### Топик не переименовывается / инфо-сообщение не закрепляется

- Дайте боту право **«Управление темами» (`can_manage_topics`)** в настройках прав администратора группы
- Без этого права бот всё равно ответит — переименование и закрепление просто пропускаются (graceful fallback)

### Веб-поиск не срабатывает

- Проверьте, что `TAVILY_API_KEY` задан в `.env` (в логах при старте должно быть `Web search: enabled (Tavily key set)`)
- Убедитесь, что модель/эндпоинт поддерживают **function/tool calling** — иначе бот отвечает без поиска (graceful fallback)
- Поиск вызывается **по решению модели**: для явно «свежих» вопросов (погода, новости, курсы) он срабатывает чаще

### "Вы не зарегистрированы"

- Попросите администратора создать токен через `/generate_token`
- Отправьте боту `/start <токен>` в личке

### "Сначала настройте эндпоинт и модель"

- Отправьте `/settings` в личке с ботом
- Добавьте эндпоинт через меню
- Выберите модель

### Ошибки при валидации эндпоинта

- Проверьте корректность URL (должен начинаться с `http://` или `https://`)
- Проверьте, что API ключ действителен
- Убедитесь, что эндпоинт доступен (не блокирован файрволом)

### Ошибки совместимости Python 3.14

- Используйте Python 3.12 для стабильной работы
- Или установите зависимости с флагом: `PYO3_USE_ABI3_FORWARD_COMPATIBILITY=1 pip install ...`

### litellm ошибки токенизации

- Fallback автоматический (оценка ~4 символа = 1 токен)
- Проверьте, что модель названа правильно (например, `gpt-3.5-turbo`, а не `gpt-3.5`)
